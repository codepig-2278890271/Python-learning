# 🐱 超级小猫 —— 代码结构分析 & 自定义指南

想自己改代码但不知道从哪里下手？这篇文档带你从头到尾拆解整个项目。

---

## 一、目录结构速览

```
project/
├── src/                 ← 所有代码都在这里
│   ├── __init__.py      ← 包初始化（自动设置搜索路径）
│   ├── main.py          ← 🧠 大脑：游戏主循环 + 状态机 + 碰撞
│   ├── settings.py      ← 🎛️ 控制面板：所有可以调的数值
│   ├── sprites.py       ← 🎨 角色工厂：每个会动的东西的定义
│   ├── level.py         ← 🗺️ 地图编辑器：平台、金币、敌人放哪里
│   └── ui.py            ← 🖥️ 画板：菜单、HUD、暂停界面
├── scripts/             ← 工具脚本（编译打包）
│   ├── build.sh
│   └── run_windows.bat
├── docs/                ← 文档
├── build/               ← 编译产物
│   └── bin/             ← 可执行文件
├── ui/                  ← 资源文件（图片、音效预留）
└── log/                 ← 运行日志
```

**数据流向**（记住这个，后面每条都套得上）：

```
__init__.py（路径设置，保证 src/ 内模块能互相 import）
          ↓
settings.py（常量）→ sprites.py（角色用）→ level.py（摆关卡）
                                              ↓
main.py ← 把上面全部串起来，一帧一帧跑
   ↓
ui.py（负责画界面文字）
```

## 二、settings.py —— 你最常改的文件

这个文件**一行逻辑代码都没有**，全是"数字和颜色"。想调什么直接改：

```python
GRAVITY = 0.8          # 重力加速度 → 改手感
JUMP_POWER = -14       # 跳跃力度 → 越大（越负）跳越高
PLAYER_SPEED = 5       # 移动速度
BULLET_SPEED = 6       # 子弹速度
STARTING_LIVES = 3     # 初始命数
COIN_SCORE = 100       # 每个金币多少分
```

**理解方式**：这个文件 = 游戏里所有"可以调的数字"的集中营。其他文件统统 `from settings import *` 引用这里的值。

## 三、main.py —— 游戏引擎

### 3.1 状态机

`self.state` 只能取 5 个值之一，就像一台自动售货机的 5 个模式：

```
MENU → PLAYING ⇄ PAUSED
          ↓
     GAMEOVER 或 WIN → MENU（重来）
```

以后想加"商店界面"？加一行 `STATE_SHOP = "shop"` 就行。

### 3.2 主循环 —— 游戏怎么"活"起来的

```python
while self.running:        # 只要不退出就一直循环
    self.handle_events()  # ① 处理输入（键盘、鼠标）
    self.update()         # ② 更新世界（移动、碰撞、物理）
    self.draw()           # ③ 画画面（背景→角色→HUD→界面）
    self.clock.tick(60)   # 锁60帧
```

每秒跑 60 遍。**改任何一个功能，本质上就是改这三个方法之一**。

### 3.3 关键方法速查

| 方法 | 做什么 | 在 main.py 行号 |
|------|--------|-----------------|
| `handle_events()` | 处理按键、鼠标点击 | 约 279 行 |
| `update()` | 更新所有精灵、碰撞检测 | 约 332 行 |
| `draw()` | 画背景、精灵、HUD、界面 | 约 391 行 |
| `handle_collisions()` | 玩家 vs 平台/金币/敌人/旗帜 | 约 171 行 |
| `build_level()` | 根据关卡数据创建精灵 | 约 109 行 |

## 四、sprites.py —— 每个角色的"身份证"

每个类 = 一个游戏角色。结构统一：

```python
class Xxx(pygame.sprite.Sprite):
    def __init__(self, ...):  # ① 出生：画外观、设初始位置
    def update(self):         # ② 每帧做什么：移动、动画
```

### Player 类（你控制的猫）

```
handle_input()  → 读键盘，改 vel_x / vel_y
apply_physics() → 加重力
update()        → 调用上面两个 + 冷却递减
```

改移动手感：找 `handle_input()`，改 `pygame.K_a` / `pygame.K_d` 对应的逻辑。
改跳跃：找跳跃部分的 `JUMP_POWER`。
改二段跳：找 `self.air_jumps_left`。

### Enemy 类（巡逻小怪）

```python
def update(self):
    self.rect.x += self.speed     # 水平移动
    if rect.x <= patrol_left:    # 撞左墙 → 调头
        self.speed = +2
    elif rect.x >= patrol_right: # 撞右墙 → 调头
        self.speed = -2
```

改敌人速度：找到 `self.speed = 2`，改成 4 就更快。

### Bullet 类（子弹）

```python
def update(self):
    self.rect.x += self.speed           # 水平飞
    self.rect.y = base_y + sin(...) * 4 # 上下波动 4 像素
```

改子弹快慢：改 `settings.py` 里的 `BULLET_SPEED`。
改波动大小：改 `BULLET_OSC_AMP`（幅度）和 `BULLET_OSC_FREQ`（频率）。

## 五、level.py —— 关卡编辑器

每个数据项含义：

```python
"player_start": (100, 480)       # 玩家出生点 (x, y)
"flag":        (3800, 440)       # 终点旗帜位置

"platforms": [                   # 平台列表
    (x, y, 宽度),                 # 每个平台：左上角坐标 + 宽度
]

"coins": [(x, y), ...]           # 金币位置

"enemies": [                     # 敌人
    (x, y, 巡逻左边界, 巡逻右边界),
]
```

**加一个新平台**：在 `"platforms"` 列表里加一行 `(500, 400, 120)` 就行。

**做第二关**：复制整个 `build_level_1()`，改名 `build_level_2()`，改坐标，然后在 `get_level()` 里注册。

## 六、ui.py —— 界面画板

纯绘制函数，不涉及游戏逻辑。每个方法 = 画一个界面：

| 方法 | 画的什么 |
|------|---------|
| `draw_menu()` | 开始画面 |
| `draw_hud()` | 游戏中的分数/生命栏 |
| `draw_pause_menu()` | 暂停菜单（含按钮） |
| `draw_gameover()` | 死亡画面 |
| `draw_win()` | 通关画面 |

改菜单文字、按钮位置、字体大小都在这个文件。

## 七、改东西速查表

| 你想改... | 去哪个文件 | 改什么 |
|-----------|-----------|--------|
| 跳更高、跑更快、重力 | `settings.py` | `JUMP_POWER` `PLAYER_SPEED` `GRAVITY` |
| 命数、分数 | `settings.py` | `STARTING_LIVES` `COIN_SCORE` |
| 按键绑定 | `sprites.py` → `handle_input()` | `pygame.K_a` 改成 `pygame.K_LEFT` |
| 角色外观 | `sprites.py` → `_draw_cat()` | 改颜色、形状 |
| 子弹快慢、波动 | `settings.py` | `BULLET_SPEED` `BULLET_OSC_AMP` |
| 关卡布局 | `level.py` | 改坐标数字 |
| 菜单文字/按钮 | `ui.py` | 改字符串、Rect 坐标 |
| 加新功能（如道具） | `sprites.py` + `main.py` | 新类 + 碰撞逻辑 |
| 加新状态（商店） | `main.py` | 加 `STATE_SHOP` + 对应处理 |

---

## 八、核心心法

整个游戏 = `settings（数值）→ sprites（角色）→ level（关卡）→ main（引擎）→ ui（界面）`，数据从左上流到右下，改一个地方就知道影响范围。
